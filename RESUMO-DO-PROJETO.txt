OBJETIVO
--------
Portfolio de Quality Assurance demonstrando habilidades práticas em
testes automatizados E2E, testes de API REST, CI/CD e documentação
de qualidade, usando dois produtos públicos como alvo de testes.


APLICAÇÕES TESTADAS
-------------------
1. SauceDemo (https://www.saucedemo.com/)
   E-commerce com bugs intencionais para estudo de QA.

2. Reqres.in (https://reqres.in/)
   API REST pública para operações de CRUD e autenticação.


NÚMEROS DO PROJETO
------------------
Testes E2E (Playwright/TypeScript):  19 testes em 8 arquivos
Testes de API (pytest/Python):        14 testes em 3 arquivos
Total de testes automatizados:        33 testes
Bug reports documentados:             6 relatórios (BUG-001 a BUG-006)
Page Objects criados:                 5 (Login, Inventory, Cart, Checkout, Complete)
Documentação técnica:                 4 docs + README + Glossário (17 termos)


STACK TÉCNICA
-------------
E2E:  Playwright + TypeScript 5 + strict mode
API:  pytest 8 + httpx + pytest-html
CI:   GitHub Actions (2 workflows independentes)
Padrão:  Page Object Model (POM)


DESTAQUES TÉCNICOS
------------------
1. TODOS os testes têm comentários em português explicando o "por quê"
   — código pensado para estudo e legibilidade de equipe.

2. Estratégia de documentação de bugs: testes de problem_user PASSAM
   documentando comportamento defeituoso, sem poluir o CI com ruído.

3. CI inteligente com filtro por paths: mudança em api/ não re-executa
   testes E2E (reduz custo de minutos no Actions).

4. Dados de teste centralizados em test-data.ts — zero magic strings
   nos testes, TypeScript garante autocompletar e detecção de typos.

5. Bug reports com análise de impacto Go/No Go — comunicação clara
   para stakeholders sobre o que bloqueia ou não um lançamento.
